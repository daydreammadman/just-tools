# AI 辅助开发使用指南

本文档说明如何使用 AI 助手来高效开发 Just Tools 的新工具。

---

## 快速开始

### 步骤 1: 准备 AI 提示词

将 `docs/AI_ASSISTANT_GUIDE.md` 文件的内容作为前置提示词（System Prompt 或上下文）提供给 AI。

**示例对话开始方式：**

```
[将 AI_ASSISTANT_GUIDE.md 的全部内容粘贴]

---

现在我需要开发一个新工具：[你的需求描述]
```

### 步骤 2: 描述你的需求

清晰地描述你想要开发的工具，包括：

- **工具名称**：例如 "Base64 编码解码器"
- **主要功能**：例如 "将文本编码为 Base64 或解码 Base64"
- **输入格式**：例如 "文本输入框"
- **输出格式**：例如 "编码/解码后的文本"
- **特殊要求**：例如 "支持双向转换"、"需要错误提示"

### 步骤 3: AI 生成代码

AI 会按照规范生成：
1. `config.ts` - 工具配置
2. `index.tsx` - 工具组件
3. `utils.ts` - 工具函数
4. 工具注册代码

### 步骤 4: 创建文件并测试

按照 AI 的指示创建文件，然后运行 `npm run dev` 测试。

## 示例对话

### 示例 1: 开发简单的文本工具

**你的输入：**
```
我需要一个 Base64 编码/解码工具：
- 工具名称：Base64 编解码
- 功能：支持文本与 Base64 之间的双向转换
- 使用 Tabs 切换编码和解码模式
- 需要输入验证和错误提示
```

**AI 将会：**
1. 创建 `src/tools/base64-converter/` 目录结构
2. 生成符合规范的 `config.ts`、`index.tsx`、`utils.ts`
3. 提供注册工具的代码
4. 所有样式使用 Tailwind CSS，不会硬编码

### 示例 2: 开发需要选项的工具

**你的输入：**
```
开发一个 JSON 格式化工具：
- 支持格式化和压缩两种模式
- 可以选择缩进大小（2空格或4空格）
- 验证 JSON 语法是否正确
- 语法错误时高亮显示错误位置
```

**AI 将会：**
1. 使用 Tabs 实现格式化/压缩切换
2. 使用 RadioGroup 或 Select 实现缩进选择
3. 实现 JSON 验证逻辑
4. 提供友好的错误提示

### 示例 3: 开发需要后端的工具

**你的输入：**
```
开发一个 IP 地址查询工具：
- 输入 IP 地址
- 调用后端 API 查询地理位置信息
- 显示国家、城市、ISP 等信息
- 支持查询当前访客 IP
```

**AI 将会：**
1. 设置 `requiresBackend: true`
2. 创建 API 调用逻辑
3. 实现加载状态和错误处理
4. 提供后端 API 实现建议（FastAPI）

## AI 生成代码的特点

AI 遵循 `AI_ASSISTANT_GUIDE.md` 规范后，生成的代码将：

✅ **样式规范**
- 使用 Tailwind CSS 类名
- 使用语义化颜色（`bg-primary`、`text-muted-foreground`）
- 响应式设计（`md:`、`lg:` 前缀）
- 不使用内联样式和硬编码颜色

✅ **组件规范**
- 使用 shadcn/ui 组件（Button、Card、Input 等）
- 统一的组件结构
- 清晰的事件处理

✅ **TypeScript 规范**
- 所有函数有返回类型
- Props 定义接口
- 不使用 `any` 类型

✅ **错误处理**
- 输入验证
- 友好的错误提示（toast）
- try-catch 包裹

✅ **功能完整**
- 清空/重置按钮
- 加载状态
- 成功提示

## 提示 AI 的最佳实践

### 1. 需求描述要具体

❌ 不好的示例：
```
做一个 JSON 工具
```

✅ 好的示例：
```
开发一个 JSON 格式化工具：
- 支持格式化和压缩
- 可选 2 或 4 空格缩进
- 验证 JSON 语法
- 复制功能
```

### 2. 明确工具分类

告诉 AI 工具属于哪个分类，这样 AI 能选择合适的图标：
- `text` - 文本处理
- `network` - 网络工具
- `conversion` - 格式转换
- `crypto` - 加密解密
- `development` - 开发工具
- `utilities` - 实用工具

### 3. 说明特殊需求

如果有特殊需求，明确告诉 AI：
- "需要实时预览"
- "支持拖拽上传文件"
- "需要复制到剪贴板功能"
- "需要历史记录"

### 4. 迭代改进

如果 AI 生成的代码不满意，可以要求调整：
- "增加一个清空按钮"
- "把输入框改成多行文本框"
- "添加一个选项来控制 [某功能]"
- "错误提示改成中文"

## 调试和问题解决

### 如果代码出错

1. **检查导入路径**：确保 `@/` 别名配置正确
2. **检查组件库**：确认 shadcn/ui 组件已安装
3. **查看控制台**：浏览器控制台会显示具体错误
4. **让 AI 修复**：将错误信息发给 AI，它会帮你修复

### 如果样式不对

AI 遵循规范后不应该出现样式问题，但如果有：
- 检查是否有硬编码样式（不应该有）
- 确认 Tailwind CSS 配置正确
- 检查响应式类名是否生效

### 如果功能不符合预期

告诉 AI 具体哪里不对：
```
输出的格式不对，我希望 [描述期望的格式]
```

```
能否增加一个功能：[描述新功能]
```

## 常见问题 FAQ

**Q: AI 生成的代码可以直接用吗？**

A: 遵循 `AI_ASSISTANT_GUIDE.md` 后，生成的代码可以直接使用。但建议先测试功能是否符合预期。

**Q: 如果需要修改样式怎么办？**

A: 告诉 AI 你想要的样式效果，它会用 Tailwind CSS 类名实现，不会硬编码。

**Q: 工具需要调用后端 API 怎么办？**

A: 在需求中说明"需要后端支持"，AI 会设置 `requiresBackend: true` 并提供 API 调用代码和后端实现建议。

**Q: 能开发复杂的工具吗？**

A: 可以！将复杂工具拆分成多个小功能，逐步让 AI 实现。

**Q: AI 生成的代码不满意怎么办？**

A: 告诉 AI 具体哪里不满意，要求修改。AI 会在保持规范的前提下调整代码。
